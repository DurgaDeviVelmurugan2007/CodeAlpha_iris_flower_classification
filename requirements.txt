Requirements and Their Role in the Iris Flower Classification Project

=> The following libraries are required to successfully run the Iris Flower Classification project. 
=> Each library plays a specific role in data handling, machine learning, visualization, and notebook execution.

# Data manipulation and analysis
pandas>=2.0.0

# Numerical computations
numpy>=1.24.0

# Data visualization
matplotlib>=3.7.0
seaborn>=0.12.0

# Machine learning and model evaluation
scikit-learn>=1.3.0

# Supporting libraries for scikit-learn
scipy>=1.11.0
joblib>=1.3.0
threadpoolctl>=3.2.0


---

Detailed Description of Each Requirement

Library	Version	Role in the Project

pandas	≥ 2.0.0	
=> Used to load and manipulate the Iris.csv dataset. 
=> It provides DataFrame structures for handling tabular data efficiently.
numpy	≥ 1.24.0	
=> Used for numerical computations and array operations. It is also used to create sample flower data for prediction.
matplotlib	≥ 3.7.0
=> Used to generate graphs and visualizations, such as feature importance plots and confusion matrix figures.
seaborn	≥ 0.12.0	
=> Used to create attractive statistical visualizations, especially the heatmap representation of the confusion matrix.
scikit-learn	≥ 1.3.0	
=> Provides machine learning tools such as dataset splitting, Decision Tree classifier, accuracy calculation, confusion matrix generation, and classification reports.
scipy	≥ 1.11.0	
=> Supports scientific and mathematical operations used internally by Scikit-learn.
joblib	≥ 1.3.0	
=> Helps in efficient computation and model persistence. Used internally by Scikit-learn for optimized performance.
threadpoolctl	≥ 3.2.0	
=> Controls thread usage and improves computational efficiency in Scikit-learn operations.
